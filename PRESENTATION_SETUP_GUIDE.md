## Formatting Guidelines

### Design Tips
- **Theme:** Choose a professional, clean theme (avoid overly busy designs)
- **Colors:** Use F1-appropriate colors (red/black/white) or team colors
- **Fonts:** Sans-serif fonts (Arial, Helvetica, Roboto) for readability
- **Font Sizes:**
  - Title: 36-44pt
  - Headers: 28-32pt
  - Body text: 18-24pt
  - Speaker notes: 12-14pt

### Visual Elements
- **Graphs:** Use high-quality, clearly labeled graphs from your Python analysis
  - Label all axes with units
  - Include legends
  - Use consistent color schemes across slides
  - Export graphs at high resolution (300 DPI minimum)

- **Images:** Use relevant F1 images (ensure proper attribution/licensing)
- **Diagrams:** Create clear flowcharts and system diagrams using Google Slides drawing tools or import from external tools

### Speaker Notes
- **Length:** 100-200 words per slide minimum
- **Purpose:** Should allow someone to present the slide without prior knowledge
- **Content:**
  - Explain all visuals
  - Provide context and examples
  - Include transitions to next slides
  - Note any questions team members should be prepared to answer

## Creating the Presentation in Google Slides

### Setup Steps

1. **Create New Presentation**
   - Go to Google Slides (slides.google.com)
   - Click "Blank" or choose a professional template
   - Rename: "F1 Tire Wear Model - Group [Your Group Number]"

2. **Share with Team**
   - Click "Share" button
   - Add all group members with "Editor" access
   - Ensure instructor has "Viewer" or "Editor" access

3. **Set Up Master Slides** (Optional but recommended)
   - Go to Slide → Edit theme
   - Customize header/footer
   - Set consistent font styles
   - Add group name/course to footer

4. **Create Slide Structure**
   - Add all slides according to the structure above
   - Add placeholder text and notes sections
   - Assign slides to team members for content development

5. **Import Graphs and Visuals**
   - Export graphs from your Python analysis as PNG or SVG
   - Use Insert → Image → Upload from computer
   - Resize and position consistently across slides

6. **Add Speaker Notes**
   - Click on the speaker notes section below each slide
   - Write detailed notes for each slide
   - Review for completeness and clarity

7. **Review and Refine**
   - Check for spelling/grammar errors
   - Ensure consistent formatting
   - Verify all graphs are readable and properly labeled
   - Test presentation flow
   - Practice with speaker notes

## Team Collaboration Tips

### Division of Work
- **Slide Creation:** Assign 2-3 slides per team member
- **Content Review:** Have different team member review each section
- **Speaker Notes:** Each person writes notes for their slides
- **Visual Design:** Designate one person to ensure consistency
- **Final Review:** Entire team reviews together

### Google Slides Collaboration Features
- **Comments:** Use for feedback and suggestions (Ctrl+Alt+M)
- **Version History:** Track changes and revert if needed (File → Version history)
- **Presenting Mode:** Practice together using present mode
- **Suggested Edits:** Use suggesting mode when reviewing others' work
